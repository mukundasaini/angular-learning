Components is building block of angular application. Component consis of
  • Html file
  • Css file
  • Type script file
  • Testing spec file
Useful links
Compnent Default Decorator:
                

import { Component } from '@angular/core';

    @Component({
   selector:'app-<component-name>',
   standalone: true,
   imports: [],
   templateUrl:'./<component-name>.component.html',
   styleUrl:'./<component-name>.component.css'
   })
Html File:
HTML File is either <component-name>.component.html or template content in @Component decorative UI logic is present on HTML file and file path is configured in .ts file
Useful links Ex:
Link html

 @Component({
    templateUrl:  './<component-name>.component.html'
 })
Internal html content 

 @Component({
    template:  '<h3> This is an tempalte attached with component file </h3>'
 })

Css File:
Css file is for applying styles for current component html content Useful links Ex:
Link css

 @Component({
   styleUrl:'./<component-name>.component.css'
   })
Multiple link css 

 @Component({
    styleUrls:  ['style1.css', 'style2.css']
   })
Inline css 

 @Component({
    styles: ['span{color:red}', '.btn-test{color:white;background-color:green;}']
 })

Default selector element

app prefix will add to your component names as slector defaultly When generate component using ng command like ng generate component 'component-name'


 @Component({
   selector:'app-component-name'
   })
User Defined Selector

You can give your name to selector in the component there is no rule like class name or filename should be same as selector name


 @Component({
selector:'user-defined'
})
Selector In Template 

 @Component({
selector: 'app-selector-in-template',
  standalone: true,
  imports:[UserDefinedSelectorComponent],
  template: `Selector in another component template example
